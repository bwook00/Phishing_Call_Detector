# 보이스피싱 RAG 실험 계획 (저장본)

## 전체 가설
1) 개인정보를 포함한 원본 데이터에서는 Naive RAG가 비교적 안정적인 검색 성능을 보인다.  
2) 동일 데이터를 개인정보 마스킹하면 검색 단서가 줄어 Naive RAG 성능이 저하된다.  
3) 마스킹 데이터에 pattern extraction, hybrid search, reranker를 적용한 Advanced RAG는 저하된 성능을 일정 수준 회복할 수 있다.

## 주차별 계획

### 1주차: 주제 선정
보이스피싱 탐지를 위한 RAG 시스템을 주제로 선정하고, 개인정보 포함 여부에 따른 검색 성능 차이를 핵심 연구 문제로 정의한다.  
또한 개인정보 마스킹으로 인해 Naive RAG 성능이 저하되는지, Advanced RAG가 이를 회복할 수 있는지를 주요 검증 목표로 설정한다.

### 2~3주차: 데이터 생성
- Corpus Dataset:
  - 보이스피싱 시나리오 300개
  - 정상 금융 통화 및 무관 문서 추가 구성 예정
- QA Dataset:
  - 보이스피싱 시나리오: 1000개
  - 실제 은행 통화 시나리오: 1000개
  - 맥락과 관련 없는 시나리오: 1000개

### 4~5주차: 개인정보 포함 데이터를 이용한 Naive RAG 구현 및 성능 확인
개인정보가 포함된 원본 데이터를 기반으로 Naive RAG를 구현하고, 검색 및 분류 파이프라인을 구성한다.  
이후 QA 데이터셋을 활용하여 기본 성능을 측정하고, 이후 마스킹 조건과 비교하기 위한 기준 성능(Baseline)을 확보한다.

### 6~7주차: 개인정보 마스킹 데이터 생성
원본 시나리오 데이터에서 이름, 주소, 전화번호, 계좌번호, 기관명, 금액 등 주요 개인정보와 식별 단서를 마스킹한 데이터셋을 생성한다.

### 8~9주차: 개인정보 마스킹 데이터를 이용한 Naive RAG 구현 및 성능 확인
개인정보가 제거된 데이터만을 사용하여 Naive RAG를 동일 조건에서 재구현하고, 1번 실험과 동일한 평가 절차를 적용한다.  
이를 통해 개인정보 마스킹이 검색 및 최종 분류 성능에 미치는 영향을 정량적으로 분석한다.

### 10~13주차: 개인정보 마스킹 데이터를 이용한 Advanced RAG 구현 및 성능 확인
마스킹된 데이터 환경에서 pattern extraction, hybrid search, reranker 등의 기법을 적용한 Advanced RAG를 구현한다.  
이후 2번 대비 성능 회복 여부를 검증하고, 1번 Baseline 수준까지 성능이 복원되는지를 중심으로 비교 분석한다.

### 14~15주차: 성과 보고 및 논문 작성
세 가지 실험 결과를 종합하여 개인정보 마스킹이 RAG 성능에 미치는 영향과 Advanced RAG의 보완 가능성을 정리한다.
